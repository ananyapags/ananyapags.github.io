Coding and Building GPU Kernels with Sasha Rush’s Puzzles (Upcoming)

Notes and thoughts while working through GPU Puzzles by Sasha Rush, an interactive notebook series that teaches GPU programming by having you write CUDA-style kernels in Numba.

Before I could really get moving, I had to answer a basic question for myself: what does it actually mean for Numba to map CUDA kernels from Python code?

At a high level, Numba takes a Python function, infers the types that would normally stay dynamic, and compiles that function into CUDA kernels that can run on the GPU.

The mental model I keep coming back to is:

Python syntax
-> LLVM IR
-> PTX (CUDA assembly)
-> GPU execution

That pipeline is part of what makes these puzzles approachable. You still get to reason about threads, indexing, and memory, but you can do it without immediately dropping into full CUDA C++ syntax.

Puzzle 1

The first puzzle is intentionally simple: add ten to every element in a vector.

Even with a toy example like that, the visual explanation is helpful because it makes the thread-to-data mapping obvious. You can see exactly how workers are assigned and why indexing logic matters.

Puzzle 2

The next step is adding vector a and vector b, then storing the result in c.

That sounds small, but it starts reinforcing the real lesson of the notebook series: GPU programming is mostly about mapping work clearly and consistently across many threads.

I’m treating this post as a running notebook for the patterns that keep showing up as the puzzles get more interesting: indexing, memory movement, synchronization, and how parallel work is actually organized on the hardware.
